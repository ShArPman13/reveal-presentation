MODULES

An Angular app is defined by a set of NgModules

We declare an NgModule by decorating our class with the “@NgModule” decorator

Every Angular app has a Root Module, conventionally named App Module, which provides the bootstrap mechanism that launches the application.

COMPONENTS

A Component controls a patch of screen called a View

A Component is identified by the “@Component” decorator that has a set of properties

SERVICES

A Service is useful to define things that can’t fit into a Component

Services are binding components

Dependency Injection

One of the fundamental concepts in Angular

DI is wired into the Angular framework and allows classes to configure dependencies that they need.

Data Binding

One of the fundamental concepts in Angular

It establishes a connection between the user interface and the underlying application
